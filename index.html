THE AIM

Have you ever wondered what happens to the information you put online? Who sees it, or even where it goes once itâ€™s online? Guise is a concept that caters to the curious who ask themselves these question.

Guise aims to promote web transparency and empower the users. Unlike other applications or browser plug-ins that simply alert the user to trackers and block them from tracking, Guise gives the user the power to not only see who is tracking them (an option available elsewhere) but also see what information the trackers are taking and what they are doing with this information.

A key part of this concept is creating an easy way for the consumer to acknowledge the darker side of the web, and web services common society take for granted. The side that takes the information uploaded and uses it for analytic purposes and other services.

This application highlights the willingness of consumers to publish information about themselves online without a second thought and the potential consequences or benefits of this.

Guise is an unbiased service that displays information clearly for the consumer, so they can view how their online behaviours can effect their online experience (e.g. targeted ads) and how this effects their privacy.

PRECEDENTS

The precedents that helped to influence the development of this concept include two popular anti-tracking browser plug-ins currently available for free, Ghostery and Do Not Track Plus. However, these two precedents focus on blocking trackers (one slightly more hands-on than the other in approach but both being competitors for the same market).

These two precedents achieve one part of the Guise aim successfully, but are still limited when it comes to allowing the consumer to see what information is being tracked, and for what purpose. This is where Guise aims to take one step further and help to educate and inform the consumer in a non-invasive manor.

THE APPLICATION

The mobile application will send pop-up notifications to the user while they browse, alerting to the trackers present. The user can then load the application to view more details, such as how many trackers currently are monitoring. From there the user can select each tracker individually to view specific information about who the tracker is, how they link to the current site, what information they are tracking and why. This is all presented to the user in clean list format with collapsable tabs.

THE FUNCTION

Guise works by monitoring the web servers called from the current web page the user is browsing, and matches it against a list of trackers. This imformation is then displayed in an easy to read and understandable manor so the user can view which trackers are monitoring their actions. This is linked to the cookies and tags websites place on the device as the user browses, and aims to highlight this to the user.

The code languages needed to achieve this include basic html and css along side bootstrap. Also, the implementation of PHP code, and jQuery/JavaScript will be required.

The code required to make this application is very complex, as is the service the application aims to offer. Extensive trialing and research is required to get the desired effect, especially as this design concept ventures into a more specific field of the web that is not often acknowledged. This means references are limited and examples of functioning code are hard to discover (making it more labour intensive).
